# app-manager — CLAUDE.md

自分が開発したアプリの進捗・バージョンを管理するWebツール。
VPS（自前サーバー）上で稼働しており、pm2 で常駐起動している。

---

## 運用ルール

### 修正前に必ず確認する
- コードを変更する前に、何をどう変えるかをユーザーに説明して確認を取ること。
- 確認なしに実装を進めてはいけない。

### 修正後に必ず行うこと（順番通りに）
1. `CLAUDE.md` を更新する（変更内容を反映）
2. ビルドを実行する: `npm run build`
3. pm2 でサーバーを再起動する: `pm2 restart next-app`

> VPSはセキュリティ上セッション接続時間を短く設定しているため、
> 修正内容を確実に反映させるためにビルド＋再起動まで一連で行うこと。

## データモデル

```ts
type AppRecord = {
  id: number;           // Date.now() で生成
  name: string;         // アプリ名
  version: string;      // バージョン番号
  createdAt?: string;   // 開発日（YYYY-MM-DD）
  updatedAt: string;    // 更新日（YYYY-MM-DD）
  status?: string;      // 開発中 / リリース済み / メンテナンス中 / 非公開
  platform?: string[];  // iOS / Android / Web / その他（複数選択可）
  storeUrl?: string;    // ストアURL（App Store / Play Store など）
  memo?: string;        // メモ（次にやること、課題など）
}
```

- `days`（更新からの経過日数）はフロントエンドで計算して表示
- データは `data/apps.json` に同期書き込み（`fs.writeFileSync`）
- 新規フィールドはすべてオプション（既存データとの後方互換性を維持）

## API エンドポイント

| メソッド | 処理 |
|---------|------|
| GET `/api/apps` | 全アプリ取得 |
| POST `/api/apps` | 新規追加（bodyに name/version/createdAt/updatedAt） |
| PUT `/api/apps` | 更新（bodyに id + 各フィールド） |
| DELETE `/api/apps` | 削除（bodyに id） |

## 主な機能

- アプリ一覧をテーブル表示（PC）/ カード表示（スマホ）のレスポンシブ対応
- アプリの追加 / 編集 / 削除（CRUD）
- ステータス管理（開発中 / リリース済み / メンテナンス中 / 非公開）をカラーバッジで表示
- プラットフォーム管理（iOS / Android / Web / その他、複数選択可）
- ストアURL登録（アプリ名クリックでストアを開く）
- メモ欄（次にやること、課題など）
- 更新日から **7日以上** 経過したアプリを赤字＋太字で警告表示
- ダッシュボード（総アプリ数・リリース済み・開発中・未更新数）
- ピンク基調のデザイン（ブランドカラー）

## 今後の方針（検討中）

### 目標
BOOTH などの販売サイトでソースコードをzip販売し、ポートフォリオとしても活用する。
ターゲットは**個人開発者・エンジニア**。

### 販売形式の選択肢（未決定）

| 形式 | ターゲット | 特徴 |
|------|-----------|------|
| **Electronアプリ** | 誰でも | .exe / .dmg をダブルクリックで起動。非エンジニアにも使いやすい |
| **Next.jsソース販売** | エンジニア限定 | 今のアーキテクチャを活かせる。DockerやREADMEを整備すればハードル下がる |
| **SaaS（Webサービス）** | 誰でも | URLを開くだけで使える。認証・DB・課金など構築コストが高い |

### 現時点の候補
- Next.jsソース販売 が現実的（今のコードベースを活かせる）
- Dockerで一発起動できるようにすると買った人のハードルが大きく下がる
- 方向性はオーナーが検討中

### 機能改善案（優先度高い順）
1. アプリのステータス管理（開発中 / リリース済み / メンテナンス中 / 非公開）
2. プラットフォーム（iOS / Android / Web）＋ ストアURL
3. ダッシュボード（総アプリ数・更新が滞っているアプリ数など）
4. 更新メモ・バージョンごとの変更履歴
5. UI刷新（Tailwind CSS など）
6. SQLite への移行（データ量が増えた場合）
